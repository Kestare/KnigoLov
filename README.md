# КнигоЛов - Библиотечная система

Веб-приложение для автоматизации работы библиотеки. 
Система позволяет управлять книжным фондом, бронированием книг, выдачей и возвратом, 
а также автоматически начислять штрафы за просрочку.
---
## Функциональные возможности

### Читателя:
- Регистрация и авторизация
- Просмотр каталога книг с поиском и фильтрацией
- Бронирование книг
- Отмена бронирования
- Личный кабинет с историей бронирований и штрафами

### Библиотекаря:
- Управление книгами (добавление, редактирование, удаление)
- Управление авторами, жанрами, издательствами
- Просмотр всех бронирований
- Оформление выдачи и возврата книг
- Управление штрафами (просмотр, отметка оплаты)
- Панель управления с общей статистикой

## Установка и запуск

### 1. Создание виртуального окружения

```bash
python -m venv venv
venv\Scripts\activate         # Windows
source venv/bin/activate      # Linux/Mac
```
### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Настройка базы данных

Создайте базу данных и настройте подключение в файле .env:

```text
DATABASE_URL=postgresql://ИМЯ_ПОЛЬЗОВАТЕЛЯ:ПАРОЛЬ@localhost:5432/НАЗВАНИЕ БАЗЫ ДАННЫХ
```
Измените в config.py:
```text
SQLALCHEMY_DATABASE_URI = 'postgresql://ИМЯ_ПОЛЬЗОВАТЕЛЯ:ПАРОЛЬ@localhost/НАЗВАНИЕ БАЗЫ ДАННЫХ'
```

### 4. Запуск приложения
#### Способ 1: Локальный запуск
```bash
# 1. Первый запуск приложения (создаст таблицы)
python app.py
# Ctrl+C - останавливаем

# 2. Заполняем данными
python populate_db.py

# 3. Снова запускаем приложение
python app.py
```
#### Способ 2: Запуск сервера с доступом по сети

###### Если требуется запустить сервер для доступа с других устройств:

```bash
#В файле app.py в самом конце убедитесь, что строка запуска выглядит так:
if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=True)
    
#Запустите сервер командой:
python -m http.server --bind 0.0.0.0 5000
```
## После запуска приложение будет доступно:

    Локально: http://127.0.0.1:5000

    По сети: http://[IP-адрес-компьютера]:5000
